Array Fundamentals

First, Arrays are a variable type

An array is a fundamental data structure in programming that allows you to store a collection of elements of the same type under a single variable name. Arrays are commonly used for organizing and manipulating data efficiently.

Yes, an array can hold a collection and all the other things that you may have heard about arrays, but first and foremost, an array is a variable type.

Recall what you know about variables, this means that an array must have a known size and set of rules, and it must address a location in memory.

How much space do I need?

All elements of an array must be the same type (and size!), and the number of elements must be known when it is declared. When the compiler identifies the space required for any variable, it needs to know how much space is required.

An array is stored in contiguous memory

The application will map any variable to an actual location in memory. Arrays are no different.

Contiguous not continuous!

Contiguous memory means that the memory allocated for an array is a single block of memory, where each element is stored in a sequential manner, one after the other, without any gaps or interruptions.

Continous means there is no end --ever. That would be amazing!

The address of an array maps to the start of the block of memory reserved to store its data.

Since each element is the same size (type), if the first element is stored at the start of the block, finding the second one is simply a matter of moving through the memory, past exactly the size of the first element. Getting to the third means starting from the beginning and going past the first two (2) elements.

We call this the index of the array element. So, myArray[5] is referring to an element located past the first 5 items counted from the beginning of the memory block, or the sixth element.

myArray[5] contains the sixth element of the array.

It seems that I have this discussion with colleagues almost every year. I've been told that hardware-guys count from one, but software-guys count from zero.

And yet, when I hold up my hand and ask how many fingers I am holding up, I get the same answer from both groups.

I maintain that this is nonsense. We, collectively, all count from the first element as one.

Don't make this more complicated than it needs to be.

An array index, or subscript, is a memory offset from the starting memory address.

Consider the image. On a whiteboard, I can easily demonstrate moving from the first element to the second and so on. But if you think about it, this is not a very efficient way for a computer to find something in memory.

Instead of "moving" to the first element, and "counting" to the next, the computer can simply calculate the address by taking the base address and adding the size of the element, since they're all the same size, and multipying by the index to get the offset.

Consider an array of integers, each 4 bytes in size:

If the array is at address 0x1000, mapped by the variable name, like any other variable. Then the first element is at that address plus zero times the size of the elements (4 bytes), so zero bytes. So the first element is at 0x1000.

The second element is at 0x1000 + (1 * 4) = 0x1004, and the third element is at 0x1000 + (2 * 4) = 0x1008.

But the computer doesn't have to "count" to find the item. To get to the two-hundred and thirteenth element, arr[212], the computer simply calculates the offset: 212 times the size of the element, which is 4 bytes, and adds that to the base address.

So when you see an array index, or subscript, it is simply a memory offset from the base address of the array.

Index = offset

And, understanding this, you can go back to counting from one, like the rest of us.

Declaring Arrays

Arrays are declared by specifying the data type of the elements they will contain, followed by the array name and the size of the array (number of elements).

Why? Because the application has to know how much space it needs.

The size of the array must be known when it is declared. Since we have to have a contiguous block of memory, the size cannot change after the array is declared.

If you need to have more elements later, the computer will not add size to the array. Instead it will create a new array in another location that is the correct size, copy the elements from the old array to the new one, and then delete the old array.

Understanding how the computer accesses the elements, makes this clearer. If the computer had to jump around in memory to find the next element, items would be slower to access or be lost.

Assigning Values to Arrays

Once declared, you can assign values to individual elements of the array using indexing. Indexing starts at 0 for the first element and goes up to the size of the array minus one.

Storing and Retrieving Elements

Elements in an array are stored sequentially in memory, allowing for fast access based on their index. You can retrieve elements by specifying their index within square brackets. 


                        # Python
                        myArray = [10, 20, 30]
                        print(myArray[0])  # Output: 10
                        print(myArray[2])  # Output: 30
                        


                        // C#
                        int[] myArray = { 10, 20, 30 };
                        Console.WriteLine(myArray[0]);  // Output: 10
                        Console.WriteLine(myArray[2]);  // Output: 30
                        


                        // Java
                        int[] myArray = { 10, 20, 30 };
                        System.out.println(myArray[0]);  // Output: 10
                        System.out.println(myArray[2]);  // Output: 30

Arrays provide a convenient and efficient way to work with collections of data in programming, making them a valuable tool for various tasks.

 

These rules for how to create and use arrays are important, but if you understand how arrays work, then the why they work will make sense, and the rules will make more sense.

 

Later, you will learn about other data structures that can be used to store collections of data, such as lists, sets, and dictionaries. Each of these has its own rules and uses, but they all share the same fundamental concept of storing data in a structured way, and understanding arrays will help you grasp these other data structures more easily.
