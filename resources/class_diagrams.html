Understanding the 3-Section UML Class Diagram

Unified Modeling Language (UML) is a standardized way to visualize software and and other engineering systems. One common diagram you'll see is the Class Diagram.

What is a UML Class Diagram?

A UML class diagram shows the structure of a class: its name, its attributes (fields), and its operations (methods). It's a powerful tool for planning and understanding your code.

The 3 Sections of a UML Class Box

Each UML class box is divided into three sections from top to bottom:

  1. Class Name: The name of the class is at the top.
  2. Attributes: The middle section lists the properties or fields of the class.
  3. Methods: The bottom section lists the functions or methods that the class can perform.

        ---------------------------
        |       ClassName         |  ← Class name section
        ---------------------------
        | - field1: Type          |  ← Attribute/Field section
        | - field2: Type          |
        ---------------------------
        | + Method1(): ReturnType |  ← Method/Operation section
        | + Method2(param: Type)  |
        ---------------------------
