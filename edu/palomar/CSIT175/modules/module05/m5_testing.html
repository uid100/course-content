Software testing

Software testing is a crucial part of the software development lifecycle. It ensures that the software meets the specified requirements, works as expected, and is free from defects. Software testing can be categorized into different phases and levels, each serving a specific purpose in the testing process.

Unit Testing

In this phase, individual units or components of the software are tested in isolation.

  • The purpose is to validate that each unit performs as designed.

Integration Testing

Integration testing focuses on testing the interactions between different units/modules/components to ensure they work together correctly. Test cases are designed to test interfaces, data flow, and interactions between modules.

  • Integration testing verifies that integrated units communicate and function as expected within the larger system.

System Testing

System testing evaluates the behavior of the entire system as a whole.

It verifies that the software meets the specified requirements and functions correctly in the intended environment. This phase may include functional testing, usability testing, performance testing, security testing, and more.

Acceptance Testing

Acceptance testing involves validating the software against the business requirements and user expectations. It ensures that the software meets the acceptance criteria defined by stakeholders.

Software engineering often refers to this as V & V, or verification and validation.

  • Verification measures performance against each individual requirement.
  • Validation assesses the system in its intended operational environment to see that the needs of the user are met

To put it another way:

Verification: did we build it right?

Validation: did we build the right thing?

Testing is an important consideration of software development and testing activities begin from the start of project. Testing activities include:

  • Reviewing requirements for testability and test planning
  • Reviewing system design for relevance against requirements and consideration of top-level objectives
  • Reviewing development process and requirements management throughout system development
  • Oversight of internal and customer acceptance testing activities
  • Regression testing and requirements management through maintenance and support after deployment
