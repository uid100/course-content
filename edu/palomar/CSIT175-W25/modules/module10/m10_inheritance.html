OOP - inheritance

In object-oriented programming (OOP), inheritance is a powerful concept that allows you to create new classes based on existing ones. This promotes code reusability and allows you to establish a hierarchical relationship between classes.

Understanding Class Inheritance

In Python, class inheritance is achieved by creating a new class (called a subclass or child class) that derives from an existing class (called a superclass or parent class). The child class inherits attributes and methods from the parent class, and it can also add its own attributes and methods or override existing ones.

Example:

Consider the following example:


            class Animal:
                def __init__(self, name):
                    self.name = name
                
                def sound(self):
                    pass
            
            class Dog(Animal):
                def sound(self):
                    return "Woof!"
            
            class Cat(Animal):
                def sound(self):
                    return "Meow!"
            
            # Create instances of child classes
            dog = Dog("Buddy")
            cat = Cat("Whiskers")
            
            # Accessing attributes and methods
            print(dog.name)  # Output: Buddy
            print(dog.sound())  # Output: Woof!
            
            print(cat.name)  # Output: Whiskers
            print(cat.sound())  # Output: Meow!

Explanation:

In this example:

  • We define a parent class Animal with an __init__ method to initialize the name attribute and a sound method that will be overridden by child classes.
  • We define two child classes Dog and Cat, each inheriting from the Animal class. Each child class defines its own sound method.
  • We create instances of each child class and demonstrate how they inherit the name attribute from the parent class and implement their own sound method.

Conclusion:

Class inheritance in Python is a fundamental concept in OOP that allows you to create hierarchical relationships between classes, promoting code reuse and organization. By understanding inheritance, you can design more flexible and maintainable software.
