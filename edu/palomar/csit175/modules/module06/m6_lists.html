We started this discussion reviewing variables in order to better understand arrays and now we'll discuss that just as Python handles variable declarations differently than other languages, data collections are also different.

While an understanding of arrays, their rules and limitations and handling, will certainly be of help, in Python we will use a List instead.

A Python List is not an array

Python Lists vs. Arrays

Lists in Python

In Python, lists are versatile data structures that can hold elements of different data types and sizes. They are mutable, meaning their elements can be changed after creation. Because the memory for an array needs to be known in order to reserve the memory for it, the size cannot change; it is immutable.

Lists in Python are declared using square brackets [ ]. For example: 

my_list = [1, 2, 3, 4, 5]

Elements in a list are indexed starting from 0. They can be accessed using square brackets and the index of the element. For example: 

print(my_list[0])  # Output: 1

Elements can be added to the end of a list using the append() method or inserted at a specific index using the insert() method. Similarly, elements can be removed using the remove() method or pop() method.

Arrays in Python

Unlike lists, arrays in Python are provided by the array module and can only store elements of the same data type. They are more memory-efficient compared to lists when dealing with large datasets of a single data type.

Arrays in Python are declared using the array() function from the array module. For example: 

import array
my_array = array.array('i', [1, 2, 3, 4, 5])

In the above example, 'i' specifies the type code for integers.

Similar to lists, elements in arrays are indexed starting from 0 and can be accessed using square brackets and the index of the element. For example: 

print(my_array[0])  # Output: 1

Arrays support many of the same operations as lists, such as appending elements using the append() method or removing elements using the remove() method.

Comparison

Lists and arrays in Python serve similar purposes but have some key differences:

  • Lists can hold elements of different data types, whereas arrays can only hold elements of the same data type.
  • Arrays are more memory-efficient compared to lists when dealing with large datasets of a single data type.
  • Lists are more versatile and offer more built-in methods for manipulation compared to arrays.
