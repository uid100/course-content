Chapter 8
  • (text) PL&D - Chapter 8

We began talking about programming in terms of detailed, step-by-step sequences of instructions. We learned about grouping those together into logical groups or chunks of code that can be called to perform specific useful tasks without using as much detail. In the last module you learned about objects and how to classify them in order to create reusable related objects.

Thinking about code in terms of objects instead of individual, discrete procedures or steps enables the programmer to create multiple smaller reusable code assemblies that can interact and do more than what otherwise becomes singular and fragile monolithic blocks of code.
