Introduction
Client-Server Architecture on IP

Web services are a way for different applications to communicate with each other over the internet using standard protocols. They allow for interoperability between different systems, enabling them to share data and functionality.

Before we start looking at code, there are several related but distinct concepts that we should take the time to investigate and define to reduce confusion.

Internet vs World Wide Web

The terms "Internet" and "World Wide Web" (WWW) are often used interchangeably, but they actually refer to different concepts.

Internet:

  • The Internet is a global network that connects millions of computers and other devices worldwide.
  • It's a massive network infrastructure that enables communication and the transfer of data between devices through various protocols such as TCP/IP (Transmission Control Protocol/Internet Protocol).
  • The Internet allows for the exchange of information, communication via email, instant messaging, video conferencing, file sharing, and much more.
  • It encompasses a wide range of technologies and services beyond just web browsing.

World Wide Web (WWW):

  • The World Wide Web is a system of interconnected hypertext documents and resources linked via URLs (Uniform Resource Locators).
  • It's a subset of the Internet and operates on top of it.
  • The WWW was launched in 1989 as a way to share documents over the Internet. It became publicly accessible in 1991.
  • Web pages on the WWW are typically written in HTML (Hypertext Markup Language) and can contain text, images, videos, hyperlinks, and other multimedia elements.
  • Users access web pages using web browsers such as Google Chrome, Mozilla Firefox, Safari, etc.
  • The WWW enables various services and activities such as web search, online shopping, social networking, blogging, and much more.

 

Web Page vs Web Site vs Web Application

Web Pages:

A web page is a single document that is part of a website. It typically contains:

  • Text
  • Images
  • Videos
  • Hyperlinks
  • Other multimedia elements

Websites:

A website is a collection of related web pages that share a common domain name. It:

  • Consists of multiple web pages
  • Is organized hierarchically
  • May be static or dynamic
  • Examples include: personal blogs, news portals, e-commerce platforms

Web Applications:

A web application is an interactive software program accessed via a web browser. It:

  • Allows users to perform specific tasks or access services
  • Can be simple or complex
  • Is built using HTML, CSS, and JavaScript for the front end, and server-side technologies like PHP, Python, or Node.js for the back end
  • Examples include: Gmail, Facebook, Google Maps
Web Servers and (REST) APIs

A web server is a software application that serves web pages to clients (usually web browsers) over the internet. It listens for incoming requests and responds with the requested resources, such as HTML files, images, or other data.

An API (Application Programming Interface) is a set of rules and protocols that allows different software applications to communicate with each other. A REST (Representational State Transfer) API is a type of API that uses HTTP requests to perform operations on resources, such as retrieving, creating, updating, or deleting data.

Web services often use REST APIs to enable communication between different applications over the internet. This allows developers to build applications that can interact with other services, access data, and perform various functions without needing to understand the underlying implementation details.

There are many servers and many services running on the internet. For now we'll focus on two.

A server delivers a predictable response to a well-formed request.

We have seen how content is commonly served on the WWW by web servers and (REST) APIs. These are also similar but different concepts.

They have a different purpose

  • REST API: Provides a way for clients to interact with a server to perform operations on resources using standard HTTP methods.
  • Web Server: Serves web content (such as HTML pages, images, CSS files) to clients over the internet.

They have different Functionality

  • REST API: Enables clients to interact with resources, such as creating, reading, updating, and deleting data, using standard HTTP methods.
  • Web Server: Serves static content and handles dynamic content generation using server-side scripting languages.

They have different Level of Abstraction

  • REST API: Operates at a higher level of abstraction, providing a standardized way for clients to access and manipulate resources over the web.
  • Web Server: Operates at a lower level, handling the mechanics of serving web content over HTTP.

They have different Interactivity

  • REST API: Enables interactive communication between clients and servers, allowing clients to request specific data or perform actions on resources.
  • Web Server: Serves static content and handles simple interactions, but does not provide the same level of interactivity as a REST API.

A REST API is commonly used for serving database content across the internet. Web servers deliver content formatted for web browser applications.
