Lab 6: Using Functions, arguments, modules

Introduction

This lab provides more opportunity to apply the concepts and principles presented in the textbook. We will be focusing on creating user-defined methods. The idea here is to build a toolbox of reusable code that you may reuse for other Python programming exercises and perhaps even in the work place.

In real-world software development, it is a common practice to implement libraries of reusable code needed for a larger project in a small test fixture environment similar to what we will be creating in some of these exercises. Code can be more easily tested/debugged in a small scale environment; different algorithms can be coded and tested to find the one with the best performance, etc. Once everything is tested and working correctly, it can then be integrated into a larger scale project and reused in future projects.

A function in Python is a unit of code that performs a task.

Start by determining what your function will do, it's purpose, whether it will return a value and all the information it will need to have.

The goal when you design and create a function should be for it to do only one thing.

Functions

You have seen how to use some of Python's built-in functions such as input(), print(), float() and others. In this lab you will practice writing your own functions.

  1. Give your function a meaningful and descriptive name. Apply the same rules as for other identifiers, such as variables, constants, and file names. If your code is easier to read, it will be improve maintainability, extensibility, and reliabilty.
  2. Determine what it will return. If you send someone outside to count the cars parked in the parking lot, do you want to find out how many cars there are, or are you just keeping them busy?
  3. What is the information it will need to do its job? These are called parameters supplied to the function.

In code, a function is easily distinguishable from a variable by the parentheses attached to it. This is the parameter list. Even it if there are no parameters being passed to the function, the empty parameter list is still contained in parentheses.

 

Define the function with the def keyword. You can follow the examples, and remember that Python is very particular about indentation.
