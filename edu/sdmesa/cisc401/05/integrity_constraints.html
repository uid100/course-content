Domain Constraints limit what kind of data you can store in a particular column.
  • Key Constraints require that every record (row) in a table needs to be unique. The primary key is a special column that ensures this.
  • NULL Constraints define whether a column can be left empty or must have a value.
  • Entity Integrity ensures that every row in a table is unique and identifiable.
  • Referential Integrity Constraints ensures that relationships between tables are valid

    • 1. Domain Constraints

    Domain constraints define the permissible values for a given column in a database table. These are the restrictions on the type, format, or range of values that can be stored in a column.

    • Data Type: Specifies the type of data (e.g., INTEGER, VARCHAR, DATE).
    • Range: Restricts values to a specific range (e.g., salary must be between 1000 and 5000).
    • Format: Ensures that values adhere to a particular format (e.g., email addresses must follow an "abc@domain.com" format).

    2. Key Constraints

    Key constraints ensure the uniqueness and identification of data within a table.

    • Primary Key: A column or a combination of columns that uniquely identify a record in the table. It cannot contain NULL values.
    • Unique Key: Ensures that the values in a column or group of columns are unique across all rows, but unlike the primary key, it can allow NULL values.

    3. NULL Constraints

    NULL constraints define whether or not a column can accept NULL values.

    • NOT NULL: Ensures that a column cannot have a NULL value; a value must be provided for every record.
    • NULL: Allows a column to have NULL values (though this is often implicit unless constrained otherwise).

    4. Entity Integrity

    Entity integrity ensures that each row in a table can be uniquely identified. This is typically enforced by a primary key, which ensures no two rows can have the same primary key value and that it cannot be NULL.

    5. Referential Integrity Constraints

    Referential integrity constraints are rules that ensure consistency between tables. They typically involve relationships between primary and foreign keys.

    • Foreign Key: A column or set of columns in one table that references the primary key in another table. This ensures that a record in one table corresponds to a valid record in the related table.
    • Cascade Delete/Update: When a record in the parent table is deleted or updated, the related records in the child table may also be automatically deleted or updated.
