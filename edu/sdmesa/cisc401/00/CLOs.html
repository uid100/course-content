CISC 401

Database Management for Healthcare Professionals

 

Course Learning Outcomes

Measurement Period: Fall 2024

CLO #1

Conduct requirement analysis of healthcare information systems.

Objective
75% of students
Assessment
(module 3) Discussion: Heathcare Data Entities (Collaborative)
Demonstrate understanding of how data is organized, stored, and managed in a healthcare database by identifying key data entities, their associated fields and attributes, and the relationships between them.
Metric
Assessment weighting:
  • (70%) Identify Key Entities, Attributes and Relationships
  • (30%) Peer Review: Provide constructive feedback on entity identification, attribute definition, and relationship description. Offer suggestions for improvement or additional insights to enhance database design, and overlap and integration opportunities with others' designs.
Results for Measurement Period
Objective: satisfactory completion of at least 75% of student submissions
  • 95% (19 of 20) students scored 70% or higher.
  • objective met
CLO #2

Create Entity-Relationship diagrams based on requirement analysis of healthcare information systems.

Objective
75% of students
Assessment
(module 4) Exercise: Create a Chen-format Entity-Relationship Diagram (ERD)
Use the concepts and symbols of the Chen ERD notation to create an Entity-Relationship Diagram for the database structure you designed in your last assignment.
Metric
Satisfactory completion of at least 75% of student submissions
Results for Measurement Period
  • 100% (20/20) students scored 70% or higher.
  • objective met
CLO 3

Use SQL to create databases, create tables, insert data into tables, and select data from tables for sample healthcare databases.

Objective
75% of students
Assessment
(module 9) Exercise: Patient Database Records
Create, populate, and query a database using SQL scripts based on the provided database schema design.
Metric
satisfactory completion consistent with rubric*
Provide SQL scripts to:
  • Create db schema with correct data types and constraints
  • Seed Database according to specification
  • Query database to retrieve specific data using joins and subqueries according to specification
Results for Measurement Period
Objective: satisfactory completion of at least 75% of student submissions
  • 76.5% (13 of 17) submissions scored 70% or higher.
  • objective met
