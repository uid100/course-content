Using table JOINs and subqueries

In SQL, data is often stored across multiple related tables, and querying data from these tables simultaneously is essential for complex database operations. Two common techniques to achieve this are JOINs and subqueries.

JOINs

A JOIN is a SQL operation that combines rows from two or more tables based on a related column, such as a shared foreign key. There are several types of JOINs, including:

  • INNER JOIN: Returns only the rows with matching values in both tables.
  • LEFT JOIN (LEFT OUTER JOIN): Returns all rows from the left table and matching rows from the right table; unmatched rows from the right are returned as NULL.
  • RIGHT JOIN (RIGHT OUTER JOIN): Returns all rows from the right table and matching rows from the left; unmatched rows from the left are returned as NULL.
  • FULL OUTER JOIN: Returns all rows from both tables, with NULLs where there are no matches.
  • CROSS JOIN: Returns the Cartesian product of two tables (all possible combinations of rows).

JOINs are used to retrieve data from multiple tables based on a related column between them. They are essential for querying complex relationships and fetching data that spans multiple tables.

Subqueries

A subquery is a query nested inside another query, often used to filter, aggregate, or calculate data.

Subqueries can:

  • Return a single value (scalar subqueries).
  • Return a list of values (used with IN or EXISTS).
  • Be used in the FROM clause as a derived table.

Subqueries are powerful when you need to:

  • Compare aggregated data to individual rows.
  • Fetch related data indirectly or with complex conditions.

Both joins and subqueries are fundamental for efficiently querying multi-table relationships, allowing you to retrieve meaningful and relevant data.
