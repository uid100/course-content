Module 3 Overview

Exercise: Create a Chen-format Entity-Relationship Diagram (ERD)

Introduction

In this exercise, you will use the concepts and symbols of the Chen ERD notation to create an Entity-Relationship Diagram for the database structure you designed in your last assignment. This exercise will help you visually represent the relationships between the entities, their attributes, and the way they interact with each other.

Note, crow's foot notation is another common way to represent ERDs, but for this exercise, we will focus on the Chen ERD style.

Instructions

Using the list of entities, attributes, and relationships you created in your last assignment, follow the steps below to generate your ERD:

1. Identify Your Entities

  • Begin by identifying the key entities from your previous work. Each entity represents a distinct object, concept, or thing in your database.
  • Entities should be drawn as rectangles in the diagram.

2. List Attributes for Each Entity

  • For each entity, list the relevant attributes that describe its characteristics.
  • Attributes are represented as ovals connected to their respective entities. At least one attribute should act as the Primary Key, which uniquely identifies each instance of the entity.

3. Indicate Primary Key

  • Select one attribute per entity to act as the Primary Key, which is an attribute that uniquely identifies each entity in the database.
  • The primary key should be underlined in your diagram.

4. Define Relationships

  • For each relationship between entities (e.g., "Works At," "Manages," "Has"), create a diamond to represent the relationship.
  • Connect the related entities to the relationship using straight lines.

5. Cardinality

  • Specify the cardinality (one-to-one, one-to-many, many-to-many) for each relationship to show how many instances of one entity can be associated with another.
  • Cardinality symbols should be added near the ends of the relationship lines (e.g., "1" for one, "N" for many).

6. Verify the Syntax

  • Make sure you use the correct symbols:
    • Rectangles for Entities.
    • Ovals for Attributes.
    • Diamonds for Relationships.
    • Lines for connections.
  • Also, ensure that the Primary Key is underlined and that cardinality is clearly indicated for each relationship.

Example Format

If you have entities like Doctor and Hospital, and a relationship such as "Works At", your ERD should indicate:

  • Doctor (Entity) ⟶ Works At (Relationship) ⟶ Hospital (Entity)
  • Attributes for Doctor such as DoctorID, Name, Specialty, etc., will be connected to the Doctor entity with ovals. Similarly, attributes for Hospital will connect to the Hospital entity.
  • Don't forget to underline DoctorID as the Primary Key.

Reminder

The Primary Key is a unique attribute within an entity that ensures each record is distinguishable from all others. Be sure to underline the Primary Key in your diagram.

Submission Guidelines

  • You may create your diagram using the Chen ERD notation using Draw.io as demonstrated in class, or a drawing tool of your choice. Be sure to save (Export) your drawing as a .jpg image.
  • Ensure all entities, attributes, and relationships are clearly labeled and connected using the correct symbols.

Grading Criteria

  • Correct use of Chen ERD notation.
  • Clear indication of entities, attributes, relationships, and cardinality.
  • Proper identification and underlining of Primary Keys.

A clear ERD can make your database structure easier to understand and implement!
