SQL Basics

Structured Query Language (SQL) is a standardized programming language used to manage and manipulate relational databases. SQL is used to create, modify, and query databases, as well as to define and enforce data integrity constraints.

Module Objectives:

  • Understand the fundamentals of databases and database management systems (DBMS)
  • Learn to design and implement relational databases
  • Gain proficiency in SQL for data manipulation and querying
  • Utilize technology for data collection, storage, analysis and reporting of information.

Some History and Background

Structured Query Language (SQL) is a special-purpose programming language designed for managing data held in a Relational Database Management System (RDBMS). SQL-like languages can also be used in Relational Data Stream Management Systems (RDSMS), or in “not-only SQL” (NoSQL) databases.

SQL comprises of 3 major sub-languages:

  1. Data Definition Language (DDL): to create and modify the structure of the database;
  2. Data Manipulation Language (DML): to perform Read, Insert, Update and Delete operations on the data of the database;
  3. Data Control Language (DCL): to control the access of the data stored in the database.

The core DML operations are Create, Read, Update and Delete (CRUD for short) which are performed by the statements INSERT, SELECT, UPDATE and DELETE.

There is also a (recently added) MERGE statement which can perform all 3 write operations (INSERT, UPDATE, DELETE).

In this module, we will have a first look at the SELECT statement, which is used to query data from a database. The SELECT statement is the most commonly used statement in SQL and is used to retrieve data from one or more tables in a database. The SELECT statement is used to select data from a database and display the results in a table.

There is a lot more to know about designing and implementing databases, but for now, we will focus on the basics of SQL and the SELECT statement.

Although the relevant tables and data are given in the examples for this module, it is important to understand that in a real-world scenario, you would have to design and implement the database yourself. This would involve creating the tables, defining the columns, and inserting the data. This requires an understanding of data constraints and relationships between tables.

Data constraints are rules that are applied to the data in a table. These rules ensure the accuracy and reliability of the data. For example, a primary key constraint ensures that each row in a table is unique, while a foreign key constraint ensures that the values in a column match the values in another table.

We'll discuss these concepts in more detail in future modules, but for now, let's focus on the basics of SQL and the SELECT statement.
