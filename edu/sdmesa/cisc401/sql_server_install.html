SQL Server

installation guide

(13-Jun-2024)

A new PC build image was used for this walk-through. No pre-existing software installations are on this (Windows 10 pro) system at the beginning of the install. These instructions and steps are subject to change, but are current as of this date.

  1. Download Installer

    Navigate to the Microsoft download link for SQL Server. Scroll down the page and find the link to download the Developer version of the product.

    https://www.microsoft.com/en-us/sql-server/sql-server-downloads

    SQL Server 2022 Developer
    is a full-featured free edition, licensed for use as a development
    and test database in a non-production environment.

    1. Launch the downloaded installation utility,
    2. Select the Basic installation,
    3. Read and accept the license terms for the developer edition of SQL Server
    4. Confirm the install location. In most cases, the default is recommended
    5. Click the [Install] to begin

      6. For reference, in my case, using a modest system configuration and relatively fast internet access, the automated installation process, including acquiring the setup files, installing and configuring the software completed in under 10 minutes.

    6. At the installation complete screen, select the option to install SQL Server Management Studio [Install SSMS]. This will open a link in your browser to download the setup utility for SSMS. After downloading the setup utility file, close the browser and launch the file (from your downloads folder.) Accept the defaul installation location and the install will typically take another 5 minutes or so.
    7. (Optional) Enable the network service.
      1. Configure SQL Server
        At this point SQL Server is installed and can be accessed from the local (Windows) machine. From the Start Menu, find the newly installed:
        "Microsoft SQL Server 2022" and
        "SQL Server 2022 Configuration" from the drop-down menu.
      2. Select SQL Server Network Configuration > Protocols for MSSQLSERVER and right-click on TCP/IP and Enable.
      3. Restart the service.
    8. The SQL Server instance will support (TCP) network access on port 1433. The final step is to configure the Windows firewall to allow access to that port. This can be accomplished quickly and simply from the command line, powershell, or management console. Refer to these instructions learn.microsoft.com
