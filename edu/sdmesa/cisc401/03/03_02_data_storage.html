Storing Data

Information

We have discussed that Data is a collection or single element of values and facts.

Information is the result of processing, manipulating, and organizing data in a way that adds value to it.

  • Information is data that has been processed and presented in a meaningful context, making it useful for decision-making and problem-solving.

If we want to access and interpret data in a meaningful way, it is important to understand how to store and manage it effectively.

Data can be classified into different types based on its format and content. Data types commonly used in databases include:

Numeric types

  • Integer ( -4, 0, 2, 106, 32762 )
    A whole number without a decimal point.
    • INT Standard integer type.
    • SMALLINT Smaller range of integer values.
    • TINYINT Very small integer values.
    • BIGINT Large integer values.
  • Float ( -10.0, 3.14, 16.666667 )
    A number with a decimal point.
    • FLOAT Single-precision floating-point number.
    • DOUBLE: Double-precision floating-point number.
    • DECIMAL or NUMERIC: Fixed-point numbers with a specific number of digits before and after the decimal point, used for exact calculations

Text or Character types

  • String ( "dude!", "one", "3-2-1-Go." "16" )
    • CHAR(n): Fixed-length string of n characters. Trailing spaces are padded.
    • VARCHAR(n) Variable-length string with a maximum length of n characters. Efficient for storing strings of varying lengths.
    • NCHAR(n) Fixed-length Unicode string of n characters.
    • NVARCHAR(n): Variable-length Unicode string with a maximum length of n characters.
    • TEXT Large amounts of text data. The exact limit varies by database system.

Date and Time types

  • DATE: Date without a time component.
  • TIME: Time without a date component.
  • DATETIME: Date and time together.
  • TIMESTAMP: A timestamp that records the date and time of an event.

Other types

There are other data types used in and by SQL variants including boolean (TRUE/FALSE), binary and abstract data types, but these are the primary data types used most commonly.

Data stored in a database must adhere to certain rules and restrictions to ensure data integrity and consistency. These rules are known as data constraints.

Common data constraints include:

  • Data Type Constraint

    Purpose: Ensures that a column contains only values of a specific data type.

    Characteristics: - Helps maintain data consistency and accuracy. - Prevents invalid data from being inserted into a column.

    Example: Defining a column as INT to store only integer values.

  • Primary Key Constraint

    Purpose: Uniquely identifies each record in a table.

    Characteristics: - Each table can have only one primary key. - The primary key must contain unique values and cannot be null.

    Example: CustomerID in a Customers table.

  • Foreign Key Constraint

    Purpose: Ensures referential integrity by linking records in one table to records in another table.

    Characteristics: - A foreign key in one table points to the primary key in another table. - Helps maintain consistency across related tables.

    Example: OrderID in an Orders table referencing CustomerID in the Customers table.

  • Unique Constraint

    Purpose: Ensures that all values in a column (or a set of columns) are unique across the table.

    Characteristics: - Unlike primary keys, unique constraints can allow null values unless explicitly restricted.

    Example: Email address in a Users table to ensure no two users have the same email.

  • Not Null Constraint

    Purpose: Ensures that a column cannot have null values.

    Characteristics: - Guarantees that a column always contains a valid value.

    Example: DateOfBirth in a Patients table, as every patient must have a birth date.

  • Check Constraint

    Purpose: Enforces domain integrity by specifying a condition that must be met for a column’s value.

    Characteristics: - Allows you to define a range or pattern for valid values.

    Example: Ensuring that Age in a Patients table is greater than or equal to 0.

  • Default Constraint

    Purpose: Assigns a default value to a column when no value is provided during record insertion.

    Characteristics: - Helps ensure that columns have a meaningful value even when omitted.

    Example: Setting a default value of CURRENT_DATE for a CreationDate column in a Records table.

  • Index Constraint

    Purpose: Enhances the speed of data retrieval operations.

    Characteristics: - Creates a data structure (index) that allows faster searching and querying of data.

    Example: Indexing the LastName column in a Employees table to speed up search queries.

  • Unique Key Constraint

    Purpose: Similar to the primary key, but allows a single null value in the column.

    Characteristics: - Can be applied to one or more columns to enforce uniqueness.

    Example: Ensuring that PhoneNumber in a Contacts table is unique but allowing null values.

  • Composite Key Constraint

    Purpose: Uses a combination of two or more columns to form a unique identifier for a record.

    Characteristics: - Ensures uniqueness when individual columns may not be unique by themselves.

    Example: A combination of OrderID and ProductID in an OrderDetails table.

  • Domain Constraint

    Purpose: Defines the permissible values for a column based on its domain.

    Characteristics: - Often enforced using check constraints or through data type definitions.

    Example: Restricting Access Leve to values such as Employee or Visitor.
