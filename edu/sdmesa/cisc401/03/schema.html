What is a Schema?

A schema in the context of databases is a blueprint or structure that defines the organization of data within a database. It specifies how data is stored, the types of data that can be held, the relationships between different data entities, and the rules for how the data can be manipulated.

Key Aspects of a Schema

Tables

A schema defines the tables within a database. Each table represents a specific entity (e.g., employees, products) and consists of columns and rows.

Columns (Attributes)

Within each table, a schema defines the columns, also known as attributes. Each column has a name and a data type, which specifies the kind of data that can be stored in that column (e.g., integer, string, date).

Relationships

A schema outlines the relationships between tables. For example, it may define foreign keys that link rows in one table to rows in another table, establishing a relationship between those entities (e.g., an order linked to a customer).

Constraints

A schema includes rules and constraints that ensure data integrity. Common constraints include primary keys, which uniquely identify each row in a table, and foreign keys, which enforce referential integrity between tables.

Indexes

The schema may also define indexes, which are structures that improve the speed of data retrieval operations by providing quick access to rows in a table.

Views

A schema can include views, which are virtual tables created by querying data from one or more tables. Views do not store data themselves but provide a way to present data in a specific format or subset.

Example

Consider a database schema for a hospital:

  • Tables: Patients, Doctors, Appointments
  • Columns:
    • Patients: PatientID, Name, DateOfBirth, Address
    • Doctors: DoctorID, Name, Specialty
    • Appointments: AppointmentID, PatientID, DoctorID, AppointmentDate
  • Relationships: The Appointments table has foreign keys PatientID and DoctorID that link to the Patients and Doctors tables, respectively.
