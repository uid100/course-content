Introduction

There is a lot of content in chapter 2. Much of it should feel like review from other languages with subtle syntax differences. Don't be in too big of a hurry, as this will lay a solid foundation and provide a good resource for quick answers until things start to stick.

Chapter 2 dives into the fundamental building blocks of C# programming: variables, operators, and expressions. It explains how to work with statements, the individual instructions that make up a program.

You should already be familiar with sequential logic, which is the basis of most programming languages. C# is no different. It executes statements in the order they are written, from top to bottom. This is called sequential logic. You will learn about the different types of statements in C# and how to use them to control the flow of your program.

Working with identifiers

A significant portion of the chapter deals with identifiers, the names you give to various elements in your code, such as variables, constants, classes, and methods.

The chapter details the rules for creating valid identifiers and perhaps doesn't go far enough to stress best practices for choosing descriptive and meaningful names. Consistent and clear naming conventions are crucial for code readability, maintainability, and collaboration, even extends to the importance of filename conventions for project organization.

The purpose and content of your files and code should be clear from the names you choose. This is a skill that develops over time, but you should always strive to make your code as self-documenting as possible. This means that someone reading your code should be able to understand what it does without needing extensive comments or documentation.

The chapter describes language keywords (reserved words with special meaning in C#) and that they are restricted from variable use, but rather than immediately memorizing a new vocabulary list, learn how you can use the IDE (Visual Studio) to help identify these while you are coding.

We use variables to store and manipulate data. Through the semester we will learn more about selecting and using data types for short and long term memory storage. For now, make note of the subtle distinction between declaring (announcing a variable's name and type), assigning (giving a variable a value), and initializing (assigning an initial value during declaration).

initialization = declaration + assignment

Data Types and Rules

  • Program Entry Point: Every C# console application begins execution from the Main method. It serves as the entry point of the program and typically resides in a class like Program.
  • Namespaces: Namespaces organize code into logical groups and prevent name conflicts. For example, System is a common namespace that provides basic functionality like input/output and string handling.
  • Using Statement: The using directive allows easy access to types in a namespace without requiring full qualification. For instance, using System; lets you use Console.WriteLine() directly instead of System.Console.WriteLine().
