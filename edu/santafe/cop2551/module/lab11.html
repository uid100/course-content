Lab 11: Understanding properties

Demonstrate ability to define and use properties in C# to encapsulate data, control access, and use backing fields appropriately.

Exercise 1: Person Profile (with Backing Fields)

Implement a class that uses explicit properties with backing fields and simple logic in the accessors.

Requirements:

  • Solution: lab11
    Project: person
  • Create a Person class with private fields: _name (string), _age (int)
  • Define public properties:
    • Name: getter returns _name, setter ensures the value is not null or empty
    • Age: getter returns _age, setter ensures value is between 0 and 120
  • Add a constructor that sets name and age, using the setter methods
  • Add a method PrintInfo() that displays name and age
  • Write a Main() method in another class to instantiate and demonstrate the Person class and use the PrintInfo() method to display the person's information
Name: Alice Age: 25 Name: Bob Age: 30

Exercise 2: Read-Only Product Info

Implement a class that uses read-only properties to expose product information.

Requirements:

  • Solution: lab11
    Project: product
  • Create a Product class with private fields: _name (string), _price (decimal)
  • Define public read-only properties:
    • Name: getter returns _name
    • Price: getter returns _price
  • Add a constructor that sets the product name and price
  • Include ApplyDiscount(double percent) method
  • Print name and discounted price in Main()

Exercise 3: Vehicle Registry with Object Initializers

Use auto-implemented properties and object initializers for creating and displaying objects.

Requirements:

  • Solution: lab11
    Project: vehicle
  • Create a Vehicle class with the following auto-properties:
    • Make (string)
    • Model (string)
    • Year (int)
    • IsElectric (bool)
  • In Main(), create at least two Vehicle instances using object initializer syntax.
  • Print a summary of each vehicle.
2022 Tesla Model 3 - Electric: Yes 2015 Toyota Corolla - Electric: No
