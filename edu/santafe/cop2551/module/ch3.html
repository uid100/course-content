Introduction

Chapter 3 introduces you to the essential building blocks of method usage in C#. You’ll learn how to define, call, and debug methods, manage data with parameters, and use features like optional parameters and named arguments.

Writing and calling methods

A method is a named block of code that performs a specific task. In C#, all functions are defined as methods, usually inside a class or struct. While the terms "function" and "method" are sometimes used interchangeably, a method specifically refers to a function that is associated with an object or class.

For example: 


class Greeter
{
    // This is a method
    public void SayHello()
    {
        Console.WriteLine("Hello!");
    }
}

Here, SayHello is a method defined inside the Greeter class. If this were a standalone function outside a class (which C# doesn't allow), it would just be a function.

Understanding Parameters vs. Arguments

In C#, parameters are variables defined in the method declaration. They act as placeholders for values the method can receive when it is called.
Arguments are the actual values passed to the method when you call it. 


// Method with parameters
void Greet(string name, int age)
{
    Console.WriteLine($"Hello, {name}. You are {age} years old.");
}

// Calling the method with arguments
Greet("Alice", 30);

In the above example, name and age are parameters. The values "Alice" and 30 are arguments.

Why Modular Code Matters

Modular code is code that is divided into reusable parts—like methods. Writing modular code makes programs:

  • Easier to read: You can understand a program by reading short method names instead of long code blocks.
  • Reusable: Once written, a method can be called from multiple places without duplicating code.
  • Testable and maintainable: Smaller pieces of code are easier to debug, test, and update.

Think of methods as building blocks. You can assemble them to build more complex features while keeping your code clean and organized.

What Is Variable Scope?

Scope refers to where a variable can be accessed in your code. In C#, variables can have different scopes depending on where they are declared:

  • Local scope: A variable declared inside a method can only be used within that method.
  • Class scope (field): A variable declared in a class but outside any method is accessible to all methods in that class.

class Example
{
    int classLevelVar = 10; // Class scope

    void ShowScope()
    {
        int localVar = 5; // Local scope
        Console.WriteLine(localVar);       // OK
        Console.WriteLine(classLevelVar);  // OK
    }

    void AnotherMethod()
    {
        // Console.WriteLine(localVar);    // ERROR: localVar not in scope here
    }
}

Understanding scope helps avoid bugs and makes your code easier to manage.

What Is a Method Signature?

A method signature in C# is the combination of a method's name and its parameter list (types, number, and order of parameters). The return type is not part of the signature.

Example of a method signature:


                      void PrintMessage(string message)
                          ↑       ↑
                      Return  Name and Parameters = Method Signature

What Is Method Overloading?

Method overloading means creating multiple methods with the same name but different parameter lists (different number, type, or order of parameters). It allows the same method name to handle different types of input.

Example of method overloading:


class Printer
{
    // Method 1: No parameters
    public void Print()
    {
        Console.WriteLine("Printing default message.");
    }

    // Method 2: One string parameter
    public void Print(string message)
    {
        Console.WriteLine(message);
    }

    // Method 3: One string and one int parameter
    public void Print(string message, int copies)
    {
        for (int i = 0; i < copies; i++)
        {
            Console.WriteLine(message);
        }
    }
}

All three methods are named Print, but they have different method signatures. The C# compiler uses the arguments you provide to determine which version to call.

Using Optional and Named Parameters in C#

In C#, optional parameters allow you to omit arguments when calling a method. These parameters have default values defined in the method signature. 


void Greet(string name = "Guest")
{
    Console.WriteLine("Hello, " + name);
}

Greet();         // Output: Hello, Guest
Greet("Alice");  // Output: Hello, Alice

Important: Optional parameters must be defined after all required parameters in the method signature.

Named Parameters

Named parameters let you specify the name of the parameter when passing arguments, which improves readability and allows you to pass arguments in any order. 


void DisplayInfo(string name, int age)
{
    Console.WriteLine(name + " is " + age + " years old.");
}

DisplayInfo(age: 30, name: "Bob");  // Output: Bob is 30 years old.

You can also combine named and optional parameters for more flexibility.

Note: Be cautious with optional parameters, as they can lead to ambiguity if overloaded methods have similar signatures. The C# compiler may not always be able to determine which method to call.
