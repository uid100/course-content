Introduction

Chapter 9 builds on your understanding of value types and considers the use of enumeration types and structure value types. In addition to being value types, so they are handled efficiently on the stack, these two user-defined types provide a way to group related data and functionality together, and more than anything else, they are useful for making your code more readable and maintainable.

Enumeration

An enumeration is a user-defined type that consists of a set of named constants. It is used to define a variable that can hold a set of predefined values, making the code more readable and maintainable.

Enumerations are defined using the enum keyword, followed by the name of the enumeration and a list of named constants enclosed in curly braces. 


        enum DayOfWeek
        {
            Sunday,
            Monday,
            Tuesday,
            Wednesday,
            Thursday,
            Friday,
            Saturday
        }

You can use an enumeration to declare a variable of that type, and assign it one of the predefined values. 


        DayOfWeek today = DayOfWeek.Monday;

        if (today == DayOfWeek.Monday)
        {
            Console.WriteLine("It's Monday!");
        }

Enumerations can also have an underlying type, which is the type of the values that the enumeration can hold. By default, the underlying type is int, but you can specify a different type using a colon. 


        enum ErrorCode : byte

As developers, we continuously strive to write code that is not only functional but also clear and maintainable. Enumerations play a crucial role in achieving this goal by providing a way to define a set of related constants that can be used to represent meaningful values in our code. They enhance the readability of our code, making it easier to understand and maintain over time.

Readability is an ongoing objective. It is a skill that gets better with practice and experience. Enumerations are a great addition to your skill set for this reason.

Structure

A structure is a user-defined value type that can contain data members and methods. It is similar to a class, but it is a value type, which means that it is stored on the stack rather than the heap.

Structures are defined using the struct keyword, followed by the name of the structure and a list of data members enclosed in curly braces. 


        struct Point
        {
            public int X;
            public int Y;
        }

You can use a structure to declare a variable of that type, and assign values to its data members. 


        Point p;
        p.X = 10;
        p.Y = 20;
        Console.WriteLine($"Point: ({p.X}, {p.Y})");

Structures can also have methods, properties, and constructors, just like classes. However, they are typically used for small data structures that do not require the overhead of a class.

Structures are useful for grouping related data together and providing a way to manipulate that data in a meaningful way. They are often used to represent simple data types such as points, rectangles, and colors.

As with enumerations, structures are a powerful tool for improving the readability and maintainability of your code. They allow you to define custom data types that can be used to represent complex data structures in a clear and concise way.
