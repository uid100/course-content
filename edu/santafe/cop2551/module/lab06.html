Lab 06: Classes and Objects

Objective: Define classes and instantiate objects.

Create classes and instantiate objects in C#. This lab will guide you through creating a simple console application that demonstrates the use of classes and objects.

The current SDK version is .NET 9.0. .NET 8.0 is the latest LTS (Long Term Support) version. Either of these versions can be used for this lab.

Exercise #1: A Book class

  1. Create the project

    Start with the solution.

    mkdir Lab06
    cd Lab06
    dotnet new sln -n Lab06

    Create a project and add it to the solution.

    dotnet new console -n BookShelf
    dotnet sln Lab06.sln add BookShelf/BookShelf.csproj

    Rename the source files to something more descriptive this time and add a new class file.

    macos:

    mv Bookshelf/Program.cs Bookshelf/Bookshelf.cs
    touch Bookshelf/Book.cs

    windows:

    ren Bookshelf\Program.cs Bookshelf.cs
    echo '' > Bookshelf\Book.cs

    Veryify the new folder structure:

    Lab06
    ├── Lab06.sln
    └── Bookshelf/
        ├── Book.cs
        ├── Bookshelf.csproj
        └── Bookshelf.cs

    In previous labs we created a single file for the source code and each was named Program.cs, which correctly matched the name of the class.

    It is best practice to name the source code file the same as the class it contains.

  2. Edit the source code

    Edit Books.cs to create a class called Book. The class should have the following members:

    This class may not match other definitions of Book objects. Use this Design for the exercise.

    Book.cs

    Do not forget to add the required header information for all of your source files!

    Notice the class "overrides" the ToString() method. This is a common practice in C# to provide a string representation of the object. The ToString() method is called automatically when the object is converted to a string, such as when it is printed to the console.

    The ToString() method should return a string that contains the details of a created object. By including this, you can format the output to display what is important to the user.

    Notice that the ToString() method returns a string, it does not print anything to the console.

    Bookshelf.cs

    Do not forget to add the required header information for all of your source files!

    The Bookshelf class contains the Main() method, which is the entry point. Notice that the Main() method:

    • public - this means that it can be called from outside the class.
    • static - this means that it can be called without creating an instance of the class.
    • void - this means that it does not return a value. Where would it return the value to?
    • Main() - this is the name of the method. It is a special method that is called when the program starts.

    This time, we have two class files, both in the same namespace, and we have renamed the source code file to match the name of the class it contains, and given the classes more descriptive names.

    There can be only one Main() method in a project.

    From the Main() method, create at least three instances of the Book class and display the output from the ToString(). The example has two. Start with these.

  3. Run and test the program

    Confirm that the program runs and instantiates your new objects and displays the details to the screen.

Exercise #3: Using static class values

  1. Create the project

    As before, add another console application project to the solution.

    • project name: ProductCatalog
    • solution name: Lab06

    Name your project ProductCatalog and add it to the solution.

    Create two class files, one for the Product class and one for the ProductCatalog class. Each of these classes should have its own source code file, named the same as the class it contains.

  2. Edit the source code

    Product

    Create the Product class according to the following class diagram:

    • Notice that the underlined field item is a static field. Initialize this to 1000
    • In the constructor:
      • Increment the static field item by one and assign the new value to the id instance field. Each item will have a unique id.
      • Set the name and price instance fields to the values passed to the constructor.
      • Set the quantity instance field to 0.
    • Create a method called ToString() that returns a string representation of the product, including the id, name, price, and quantity in stock.

    ProductCatalog

    Create a ProductCatalog class containing a Main method.

    In this class, create at least five products, each with a unique name, and price.

    Add products to the catalog and display the details of each product.

    Test the SellItem() method. It should return false if the quantity requested is more than the quantity in stock, otherwise it should reduce the quantity in stock by the quantity sold and return true.

    Display the details of each product after selling an item.

  3. Run and test the program

    Run the program and demostrate product (objects) created, added to stock and sold.
