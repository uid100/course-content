Lab 06: Classes and Objects

Objective: Define classes and instantiate objects.

Create classes and instantiate objects in C#. This lab will guide you through creating a simple console application that demonstrates the use of classes and objects.

  1. Create the project

    It is best practice to name the source code file the same as the class it contains.

  2. Edit the source code

    Edit Books.cs to create a class called Book. The class should have the following members:

    This class may not match other definitions of a Book object, but it is sufficient for this exercise.

    Book.cs

    Do not forget to add the required header information for all of your source files!

    Notice the class "overrides" the ToString() method. This is a common practice in C# to provide a string representation of the object. The ToString() method is called automatically when the object is converted to a string, such as when it is printed to the console.

    The ToString() method should return a string that contains the details of a created object. By including this, you can format the output to display what is important to the user.

    Notice that the ToString() method returns a string, it does not print anything to the console.

    Bookshelf.cs

    Do not forget to add the required header information for all of your source files!

    The Bookshelf class contains the Main() method, which is the entry point. Notice that the Main() method:

    • public - this means that it can be called from outside the class.
    • static - this means that it can be called without creating an instance of the class.
    • void - this means that it does not return a value. Where would it return the value to?
    • Main() - this is the name of the method. It is a special method that is called when the program starts.

    This time, we have two class files, both in the same namespace, and we have renamed the source code file to match the name of the class it contains, and given the classes more descriptive names.

    There can be only one Main() method in a project.

    From the Main() method, create at least three instances of the Book class and display the output from the ToString(). The example has two. Start with these.

  2. Design and Write the source code

    Moving across an X-Y grid

    Write a program that simulates moving across an X-Y grid. The program should start at the middle of the plot at position (0, 0) and keep track of the (X, Y) position. The tracker will move ten times, moving in one of four directions: up, down, left, or right. The program should display the new position after each move.

    Don't forget to add the required header information described in the previous assignments.

    Write a method called Move() and call it ten times. The method should take two parameters: the current X and Y position and return the new X and Y position as a tuple.

    You may use a loop to call the method if you prefer, or you can keep it very simple as I did in the example.

    Your code might look something like this:

    The method should randomly select one of the four directions to move in, Up, Down, Left, or Right, and also randomly select the number of steps, between 1 and 10, to move in that direction. The method should then calculate and return the new X and Y position.

    Each time the method is called, display the direction and number of steps to move.

    Up, should increase the Y position by the number of steps, Down, should moved along the Y axis in the negative direction. Left and Right should move positive and negative along the X axis.

    Finally, calculate and display the total distance traveled. And the final distance from the starting position.

    The total distance traveled is the sum of all the steps taken.

    Use the Pythagorean theorem to calculate the distance from the starting position. The formula is distance = sqrt((x2 - x1)^2 + (y2 - y1)^2).
