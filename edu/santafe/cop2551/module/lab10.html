Lab 10: Creating interfaces & defining abstract classes

Create and implement interfaces and and abstract classes. Test and compare their behavior in different scenarios.

Exercise 1: IPlayable Interface

In this exercise, you will create an interface called IPlayable and implement it in various classes as described below.

  1. Create the project

    • Solution: Lab10
    • Project: Media

  2. Design and write the code

    IPlayable.cs

    Create the interface IPlayable with the following members:

    • void Play() - Starts playback
    • void Pause() - Pauses playback
    • void Stop() - Stops playback

    Media classes

    Create the following media classes that implement the IPlayable interface:

    • Song.cs
    • Video.cs

    Player

    Create a class to contain your Main method.

    In the Main method, create a list of IPlayable items, including songs and videos.

    Display your list, including details from each of the ToString() methods, allow the user to make a selection and call Play(), Pause(), and Stop() on each object.

  3. Test the classes

    Run your program and test the functionality of the IPlayable interface and the classes that implement it.

Exercise 2: abstract Employee

Demonstrate inheritance and polymorphism by implementing different types of employees who calculate their salary differently.

Build a payroll system for a company where all employees share some common information, but different types of employees calculate their salaries in different ways.

Use an abstract class Employee to define common behavior and require derived classes to implement salary calculation.

  1. Create the project

    Create a new project called Payroll in the same solution as the previous exercise.

    • Solution: Lab10
    • Project: Payroll

    Each class should be in its own file. The file names should match the class names.

  2. Design and create the source code

    1. Create an abstract class named Employee with:
      • Fields: Name, EmployeeID
      • Constructor to set these fields
      • Abstract method: decimal CalculateSalary()
      • Method: void DisplayInfo() – prints name, ID, and salary by calling CalculateSalary()
    2. Implement both concrete subclasses with appropriate constructors and method overrides.
      • SalariedEmployee
        • Additional field: MonthlySalary
        • Implements CalculateSalary() to return MonthlySalary
      • HourlyEmployee
        • Additional fields: HourlyRate, HoursWorked
        • Implements CalculateSalary() to return HourlyRate * HoursWorked
    3. In Main():
      • Create a list of Employee objects.
      • Add instances of SalariedEmployee and HourlyEmployee to the list.
      • Call DisplayInfo() on each employee in the list.

  3. Test the classes

    Run your program and test the functionality of the Employee class and its derived classes. Ensure that each employee's information is displayed correctly and that the salary calculations are accurate.
