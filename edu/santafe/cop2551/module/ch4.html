Introduction

Chapter 4 introduces the concept of decision statements in C#.

So far, we've focused on sequential processing—writing instructions that the computer follows one after another, step by step. This approach helps build a strong foundation for understanding how programs work. But real-world problems often require more than just doing the same thing every time. What if we only want something to happen if a certain condition is met?

In this chapter, we introduce conditional processing, which allows our programs to make decisions.

You have probably heard that computers only know about 1s and 0s. Strictly speaking, this is not exactly true.

In fact, the computer does not know about 1s and 0s either. It only knows about voltages and currents. The electronics can respond to differences in voltage and currents. The 1s and 0s are for our benefit. They're just a more convenient way to represent the two states of a switch.

At the heart of conditional logic is binary thinking —something is either true or false, on or off, yes or no.

Computers are built on this concept. In electronics, transistors act like switches that either allow electrical current to pass through (on) or block it (off). We represent these two states with 1s and 0s.

While the computer doesn't "understand" what a 1 or 0 means, this binary system gives us a powerful and consistent way to model decisions in code. Using if-statements and other conditional structures in C#, we can tell the computer: "Only do this if a certain condition is true."

Conditional logic is a key building block in programming, enabling us to create more dynamic, responsive, and intelligent programs.

Boolean expressions and operations are the foundation of decision-making in programming. They're essential for writing if statements, loops, and other control structures that help programs react to different situations.

What is a bool Variable?

In C#, a bool (short for Boolean) is a primitive data type that can hold one of two values: true or false.

This is useful for representing simple yes/no or on/off conditions. For example: 


bool isGameOver = false;
bool hasPermission = true;
bool isRaining = false;

Boolean values are often used in conditional statements to control the flow of a program: 


if (hasPermission)
{
    Console.WriteLine("Access granted.");
}

Behind the scenes, the computer represents stores these binary data values which are represented in simple terms as 1s and 0s. true, typically as 1, and false as 0.

This binary representation allows computers to perform logical operations efficiently.

Boolean Expressions

A Boolean expression is any expression that results in a bool value.

While bool variables can be useful, any variable type may be used to evaluate a Boolean expression.

For example, the doorman at a nightclub doesn't need to know how old you are, just whether you are old enough to enter. so age >= 21 is a Boolean expression that evaluates to true or false, not the actual numerical value.

If a parking spot is reserved for "John" then name == "John" is a Boolean expression that evaluates whether or not the car's operator can park there.

In both of these cases, the variables being evaluated are not bool types, but the expressions themselves are Boolean.

Here are some examples of Boolean expressions: 

int age = 20;
bool isAdult = age >= 18; // true
bool isTeen = age >= 13 && age <= 19; // false

These expressions often use comparison operators:

Logical (Boolean) Operators

These operators let you combine multiple Boolean values or expressions:

Operator Name Description Example
&& Logical AND True if both sides are true true && false → false
|| Logical OR True if either side is true true || false → true
! Logical NOT Flips the value: true to false, and vice versa !true → false

Example with all three:

bool hasKey = true;
bool knowsPassword = false;

if (hasKey || knowsPassword)
{
    Console.WriteLine("You can enter.");
}

bool isLocked = !(hasKey || knowsPassword); // NOT operator

Boolean Conditional Control Statements in C#

Conditional control statements let a program choose different actions depending on whether a condition is true or false. These conditions are built with Boolean expressions.

1. if Statement

The simplest form of conditional logic. It runs a block of code only if a condition is true.

Syntax:

if (condition)
{
    // Code to run if the condition is true
}

Example:

int score = 90;
if (score >= 80)
{
    Console.WriteLine("You passed!");
}

2. if-else Statement

Provides an alternate block of code to run if the condition is false.

Syntax:

if (condition)
{
    // Code if true
}
else
{
    // Code if false
}

Example:

int score = 70;
if (score >= 80)
{
    Console.WriteLine("You passed!");
}
else
{
    Console.WriteLine("You did not pass.");
}

3. if-else if-else Chain

Used when there are multiple conditions to check in sequence. The program will run the block for the first true condition it finds.

Syntax:

if (condition1)
{
    // Code if condition1 is true
}
else if (condition2)
{
    // Code if condition2 is true
}
else
{
    // Code if none of the above are true
}

Example:

int score = 65;
if (score >= 90)
{
    Console.WriteLine("Grade: A");
}
else if (score >= 80)
{
    Console.WriteLine("Grade: B");
}
else if (score >= 70)
{
    Console.WriteLine("Grade: C");
}
else
{
    Console.WriteLine("Grade: D or F");
}

These conditional structures help programs make decisions and respond to different situations dynamically.

The switch Control Structure in C#

The switch statement in C# provides a way to select one of many possible code blocks to run, based on the value of a single variable or expression. It is an alternative to using many if-else if statements when checking a variable against multiple constant values.

Basic Syntax:

switch (expression)
{
    case value1:
        // Code to execute if expression == value1
        break;
    case value2:
        // Code to execute if expression == value2
        break;
    // More cases...
    default:
        // Code to execute if no case matches
        break;
}

Example:

int day = 3;

switch (day)
{
    case 1:
        Console.WriteLine("Monday");
        break;
    case 2:
        Console.WriteLine("Tuesday");
        break;
    case 3:
        Console.WriteLine("Wednesday");
        break;
    case 4:
        Console.WriteLine("Thursday");
        break;
    case 5:
        Console.WriteLine("Friday");
        break;
    default:
        Console.WriteLine("Weekend");
        break;
}

Important Notes:

  • The value used in case labels must be a constant or literal.
  • Each case typically ends with a break statement to prevent fall-through (executing the next case).
  • The default block is optional but useful for handling unexpected values.

The switch statement makes your code cleaner and more readable when checking a single variable against many fixed options.
