Lab 09: using class inheritance

Objective: extend base classes to derived classes in C#.

Design and implement a set of classes that demonstrate inheritance in C#.

Remember the principle of Don't Repeat Yourself (DRY). Inheritance allows you to create a base class with common properties and methods, and then derive new classes that inherit those properties and methods. This is intended to reduce code duplication and makes your code more maintainable. If you find yourself repeating code in multiple classes, consider creating a base class that contains the common code and then derive the other classes from that base class.

In this exercise, you will create a base class called Vehicle and derived classes as described below.

The UML diagram shows the inheritance between the classes. Inheritance is shown by the arrow pointing from the derived class to the base class. The derived classes inherit the properties and methods of the base class, allowing for code reuse and polymorphism. The arrow always points from the derived class to the base class indicating that the derived class contains its members as well as those of the base, not the other way around.

In documentation, you will see the terms base class and derived class. Sometimes you will see the term superclass and subclass or parent class and child class. These terms are interchangeable and refer to the same relationship.

Exercise 2: Shapes

In this exercise, create a base class called Shape and derived classes Circle, Rectangle, and Triangle.

  1. Create the project

    Create a new project called Shapes in the same solution as the previous exercise.

    • Solution: Lab09
    • Project: Shapes

    Each class should be in its own file. The file names should match the class names.

  2. Design and create the source code

    Base Class

    • Create an abstract class named Shape. This class will serve as the blueprint for all geometric shapes in our hierarchy. It should contain common properties and an abstract method GetArea()for calculating the area.
    • Add a public property for the Name of the shape (e.g., "Circle", "Rectangle").
    • Define an abstract method called GetArea() that returns a double. This method will have no implementation in the base class, forcing derived classes to provide their own.

    Derived Classes

    Create derived classes for each specific shape: Circle, Rectangle, and Triangle.

    Each derived class should implement the GetArea() method to calculate the area of the specific shape.

    Optionally:

    • Add more derived shape classes such as Square or Ellipse.
    • Implement a method GetPerimeter() in the base class (abstract or virtual depending on whether all shapes have a common perimeter calculation).

  3. Test the classes

    Create a test driver file in the Shapes project. In the Main method, create instances of each derived class and demonstrate their functionality by calling their methods and displaying their properties.

    Optionally:

    • Create a list of shapes and write a method to calculate the total area of all shapes in the list.
    • Use the Debug.Assert() method to verify your results against your calculated expected values.

Exercise 3: Product Catalog

In this exercise, you will create a base class called Product and derived classes for different types of products.

  1. Create the project

    Create a new project called ProductCatalog in the same solution as the previous exercises.

    • Solution: Lab09
    • Project: ProductCatalog

  2. Design and create the source code

    Base Class

    • Create a base class named Product.
    • Add properties for common fields such as Name, Price, and Description.
    • Define a method ToString() to display product details.

    Derived Classes

    • Create derived classes for specific product types, such as Electronics, Clothing, and Food.
    • Each derived class should have additional fields and methods specific to that product type. This part may require some creative thinking.
    • Override the ToString() method in each derived class to display the specific details of the product.
  3. Test the classes

    In exercise 1 above, you used the List<T> data structure. This data type is a much higher level data structure, but its usage is straightforward and similar to arrays. In this exercise, you may use the List<Product> structure, or you may create an array of products (Product[]).

    Create a ProductCatalog class to instantiate the derived classes and test their functionality.

    In the Main method, create instances of each derived class and demonstrate their functionality by calling their methods and displaying their properties (using the ToString() method).
