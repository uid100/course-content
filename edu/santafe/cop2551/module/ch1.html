Introduction

C# (pronounced "C-sharp") is a modern, object-oriented programming language developed by Microsoft as part of its .NET platform. It is designed for building a wide range of applications, from desktop and web to mobile and cloud-based solutions. C# combines the power of C++ with the simplicity of Visual Basic, offering a versatile and developer-friendly syntax.

Like other languages, C# is continuing to evolve and improve, with strong guidance from the community.

IDE

Visual Studio

Visual Studio is an integrated development environment (IDE) from Microsoft that provides developers with a comprehensive set of tools for building applications. It supports multiple programming languages, including C#, and offers features like code editing, debugging, and project management.

An IDE is more than just a text editor, or a compiler. It is a complete environment for writing, testing, and debugging code. It includes a text editor, a compiler, a debugger, and other tools that help you write and test your code.

Don't be overwhelmed by the number of features in Visual Studio. You will not use all of them at once. You will learn to use them as you need them. The IDE is designed to help you write code faster and more efficiently.

To start using C#, you need to install Visual Studio. The Community Edition is free and suitable for individual developers and small teams. Once installed, you can create a new project, select a template (e.g., Console App), and begin coding.

Key Concepts

  • Program Entry Point: Every C# console application begins execution from the Main method. It serves as the entry point of the program and typically resides in a class like Program.
  • Namespaces: Namespaces organize code into logical groups and prevent name conflicts. For example, System is a common namespace that provides basic functionality like input/output and string handling.
  • Using Statement: The using directive allows easy access to types in a namespace without requiring full qualification. For instance, using System; lets you use Console.WriteLine() directly instead of System.Console.WriteLine().

Developer Best Practices

Writing clear, concise comments is a key part of professional development. Use // for single-line comments and /* */ for multi-line comments. Good comments explain why code exists or how complex logic worksâ€”not what the code is doing, if it's already clear.

Comments should be used to explain the purpose of the code, not to restate what the code does. For example, instead of writing: 

// This code adds two numbers
int sum = a + b;

You should write: 

// Calculate the sum of two numbers
int sum = a + b;

The extra explanation in the example code is for student benefit. For now, you should add any comments that help you understand the code. As you gain experience, you will learn to balance the need for comments with the need for clean, readable, self-documenting code.
