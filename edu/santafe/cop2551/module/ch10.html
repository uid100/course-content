Introduction

Chapter 10 introduces arrays, a fundamental data structure that allows you to store and manipulate collections of data.

Arrays are used to store multiple values of the same type in a single variable, making it easier to manage and manipulate large sets of data. They are particularly useful when you need to perform operations on a collection of items, such as sorting, searching, or iterating through a list of values.

Specifically, the chapter highlights:

  • Declaring arrays
  • Creating an array instance
  • Storing and accessing data within arrays
  • Copying arrays, and
  • Multi-dimensional arrays

This is an important chapter, as arrays are a basic concept used widely in programming. Take the time to read the chapter and review the other resources provided in Canvas.

Having a strong understanding of arrays will help you write more efficient and organized code, especially when dealing with large datasets or complex data structures.

Below are -briefly- the points that I think are the important takeaways from the chapter:

Declaring and initializing arrays

Notice that in C#, arrays can be declared in a few different ways. All are valid, and all have their own use cases. The situation will determine which is best. Sometimes, you may know the size of the dataset but not the values, while other times you may know the values but not the size.

While reading through the material, consider when and why you might use one or another of these.

Accessing array values

An array index or subscript is used to identify a specific element stored in an array. The index is an integer value that represents the position of the element within the array.

Be careful not to confuse the index with the value stored in the array.

The index is used to access the value, but it is not the value itself.

For example, if you have an array of integers, the index might be 0, but the value at that index could be 42. The index is simply a way to reference the position of the value within the array.

Copying arrays

Copying arrays is a common operation in programming, and C# provides several methods to do this.

The common terms used when copying arrays are shallow and deep copies. A shallow copy creates a new array that references the same elements as the original array, while a deep copy creates a new array that contains copies of the elements in the original array.

Read that again.

I was surprised that the book did not mention these, but they are in common use so I wanted to make sure you are aware of them.

Remember the difference between reference and value types. An array is a reference type, meaning that when you copy an array, you are copying the reference to the array, not the actual elements. This means that if you modify the elements of the copied array, you are also modifying the elements of the original array.

This is an important distinction and you need to be careful especially when passing arrays to methods or returning them from methods.

If you want to manipulate the elements of an array without affecting the original array, you can create a deep copy of the array. That is to make a new array and copy each of the elements to the new array. This way, you can modify the elements of the new array without affecting the original array.

Multi-dimensional arrays

Multi-dimensional arrays are arrays that have more than one dimension. This can mean two or more. Although most of the examples you see will deal with two-dimensional arrays, you can create arrays with any number of dimensions.

Again, the textbook's introduction is good, but it doesn't go too deep into the topic. The author does explain the distinction between multi-dimensional arrays and jagged arrays.

Jagged arrays (sometimes called "ragged" arrays) are also known as arrays of arrays. That is to say, they are arrays that contain other arrays as their elements, which may contain different numbers of elements. This allows for more flexibility in storing data, as each sub-array can have a different length.

In contrast, multi-dimensional arrays are more structured. Every row or column or dimension has the same number of elements. This means that all rows have the same number of columns, and all columns have the same number of rows.

Once again, which type you use will depend on the situation. Jagged arrays are useful when you need to store data that has varying lengths, so that you can save memory and avoid wasting space. Multi-dimensional arrays are useful when you need to store data in a structured way, such as in a table or grid format.
