Notice that the folders are numbered "lab01", "lab02", and so on. If you number these using only a single digit, then when we labs 10, 11, and 12, the natural ordering of your folders will be lab1, lab10, lab11, lab12, lab2, lab3, ...

This is because the computer sorts the folders in "alphabetical" order, not "numerical" order. So, lab1 comes before lab10, and lab2 comes after lab10.

To avoid this problem, use 2 digit numbers for the lab folders. This way, when we get to lab10, the folders will be sorted in the correct order: lab01, lab02, lab03, lab04, lab05, lab06, lab07, lab08, lab09, lab10, lab11.

Exercise #1: Multiply

  1. Create the project

    Follow the steps in chapter to for:
    Beginning programming with the Visual Studio 2022 environment
     to create a new project.

    Use the template drop-down filters for language: C# and type: Console. You can leave platform settings to All.

    Select the Console Application template and click Next.

    Set the folder (directory) to your new lab folder (Lab01), Name your project multiply.

    Uncheck the box for "Place solution and project in the same directory" if it is checked and name the solution Lab01. This will enable you to keep all of the lab projects in the same solution. Click Next.

    Check the box for "Do not use top-level statements" and click Create.

    In the Solution Explorer within Visual Studio, you will see a solution folder named Lab01 and a project folder named multiply.

    Right-click the solution name (Lab01) and choose Open Folder in File Explorer (or Finder in macOS). This will open the folder where your solution is stored. The folder will contain a solution (.sln) file, and a project folder. The project folder will have a Program.cs file and a .csproj file, and new subfolders for binaries and object files. You can close the File Explorer for now. We'll look at it again later.

    The project .csproj file is the project file that contains information about the project, such as the name, version, and dependencies.

    The Program.cs file is the main file that contains the code for the application.

     

    Run this code to see the output. You can start it by clicking the green arrow (Start button) in the toolbar or by pressing Ctrl + F5. Or you can use either of the Start options in the Debug menu.

    You have successfully created your first C# console application without Writing any code!

  2. Edit the source code

    Open the Program.cs file in the editor. You will see a lot of code that you did not write. This is the code that was generated for you by Visual Studio.

    Notice there are three "blocks" of code in the file.

    The first block is the namespace. It is named the same as the project.

    Inside this, the second block is the class Program. This is the main class of the program. It contains the entry point of the program and it is contained in the namespace.

    The third block is the method Main(). This is the main method of the program. It is where the program starts executing.

    When you run the program, it will execute the code in the Main() method and print the output, currently "Hello World!" to the console.

    Use the editor to replace all the code with the following: 

    using System;

class Program
{
    static void Main()
    {
        // Declare variables
        int number1;
        int number2;
        int product;
        string userInput;

        // Prompt user for input (note the space after the colon)
        Console.Write("Enter the first number: ");
        // Read user input, convert to int, and assign to a variable.
        userInput = Console.ReadLine();
        number1 = Convert.ToInt32(userInput);

        Console.Write("Enter the second number: ");
        userInput = Console.ReadLine();
        number2 = Convert.ToInt32(userInput);

        // assign the value of the product on the right to the variable on the left
        product = number1 * number2;

        // Output the result. The $ sign allows you to use variables in the string.
        Console.WriteLine($"The product of {number1} and {number2} is {product}.");
    }
}

    The comments are there to explain what the code is doing. You can remove them if you want. The using System; line is a directive that tells the compiler to use the System namespace. This is where the Console class is defined.

    Our simple program will prompt for two numbers. The result of multiplying the two numbers is called the product.

    Use of number1, number2, and product are examples of variables with intentional names. The names are descriptive and meaningful. They are not just random letters or numbers. This is a good programming practice.

  3. Run and test the program

    Run the program by clicking the green arrow (Start button) in the toolbar or by pressing Ctrl + F5. Or you can use either of the Start options in the Debug menu.

    Enter two integer numbers when prompted. The program will multiply them and display the result. Observe that the output result is formatted with the numbers you entered and that the displayed result is the expected product of the two numbers.

    Try entering some different numbers. What happens if you enter a decimal number? What happens if you enter a letter or a word? What happens if you enter a very large number?

Exercise #2: Km to Miles

  1. Create the project

    Use Visual Studio to create another console application project. This application will ask the user to enter a distance in kilometers and then convert it to miles.

    This time, right-click the solution name (Lab01) in the Solution Explorer and choose Add > New Project.

    Name it kmToMiles. It is important to use meaningful descriptive names for all your identifiers, including project names and variable names and constants, and soon classes and methods. mylabex is not a good name for a project because it gives no indication of what the project is about. This will create a new project in the same solution folder.

    Right-click the new project name (kmToMiles) in the Solution Explorer and choose Set as Startup Project. Then, when you use the Run button, this project will run instead of the previous one. If you want to switch back to run a different project, you can right-click the project name and choose Set as Startup Project again, or you can choose Debug > Start Without Debugging from the menu to start another project without changing the startup project.

  2. Edit the source code

    Open the Program.cs file in the editor.

    Replace all the code with your solution.

    1 kilometer = 0.62 miles.

    This is the conversion factor that you are given. It must be in your code. Do not lookup or calculate another. Using other values will produce unexpected results.

    Note: The formula for converting kilometers to miles is:
    miles = kilometers * MILES_PER_KM

    Note: The constant MILES_PER_KM is a constant variable that holds the value of 0.62. It is a good practice to use constants for values that do not change in your program.

    It is also common practice to use all uppercase letters for constant identifiers (names).

    In the last exercise, you created integer variables to hold the values of the user inputs and the result. In this exercise, use double variables to store inputs, outputs and constants.

    To convert a string to a double, use the Convert.ToDouble() method.

    Give consideration to the names of your variables. Use descriptive names that indicate what the variable is used for. For example, km is a good name for a variable that holds the value of kilometers.

    Avoid using magic numbers (unnamed constant values) in your code. For example, instead of using 0.62 in your code, use a constant variable with a meaningful name, such as MILES_PER_KM. This makes your code easier to read and understand. It also makes it easier to change the value of the constant in the future if needed.

  3. Run and test the program

    Run the program by clicking the green arrow (Start button) in the toolbar or by pressing Ctrl + F5. Or you can use either of the Start options in the Debug menu.

    Enter a distance in kilometers when prompted. The program will convert it to miles and display the result. Observe that the output result is formatted with the number you entered and that the displayed result is the expected conversion of kilometers to miles.

    Try entering some different numbers. What happens if you enter a decimal number? What happens if you enter a letter or a word? What happens if you enter a very large number?

    What happens if you enter a negative number? What happens if you enter zero?
