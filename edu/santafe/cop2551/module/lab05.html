Lab 05: Exception Handling

Objective: Use try-catch-finally blocks to handle exceptions in C# code.

In this lab, you will practice using exception handling in C# code. You will learn how to use try-catch-finally blocks to handle exceptions and how to create custom exceptions.

Exercise #1: Safe Division with Exception Handling

  1. Create the project

    Lab05
    ├── Lab03.sln
    └── Division
        ├── Division.csproj
        └── Program.cs

    Objective

    Use a try-catch block to handle potential exceptions when dividing two numbers entered by the user, including handling division by zero and invalid input.

    Instructions

    1. Ask the user to enter two numbers: a numerator and a denominator.
    2. Convert the input to integers using int.Parse() (or Convert.ToInt32()).
    3. Perform division: numerator / denominator.
    4. Use try-catch to:
      1. Catch DivideByZeroException and display a message.
      2. Catch FormatException for invalid input and display a message.
      3. Catch any other exceptions and display a generic error message.
    5. Display a farewell message at the end using a finally block.

    Example Output

    Case 1: Valid Input 

            Enter numerator: 10
        Enter denominator: 2
        Result: 5
        Thank you for using the program!

    Case 2: Division by Zero 

            Enter numerator: 10
        Enter denominator: 0
        Error: Division by zero is not allowed.
        Thank you for using the program!

    Case 3: Invalid Input 

            Enter numerator: ten
        Enter denominator: 2
        Error: Invalid input. Please enter a valid number.
        Thank you for using the program!

    Don't forget to add the required header information.

  2. Explanation

    Understanding the code

    The code uses a try-catch block to handle exceptions that may occur during the division operation. The try block contains the code that may throw an exception, while the catch blocks handle specific exceptions that may occur.

    The first catch block handles the DivideByZeroException, which occurs when the denominator is zero. The second catch block handles the FormatException, which occurs when the input cannot be converted to an integer. The third catch block handles any other exceptions that may occur.

    The finally block is executed regardless of whether an exception occurs or not. It displays a farewell message to the user.

    The program uses int.Parse() to convert the input to an integer. If the input is not a valid integer, a FormatException will be thrown.

    The program also uses Console.ReadLine() to read input from the user. If the user enters a non-numeric value, a FormatException will be thrown.

  3. Run and test the program

    Run the program by clicking the green arrow (Start button) in the toolbar or by pressing Ctrl + F5. Or you can use either of the Start options in the Debug menu.

    Confirm that the program runs and assigns random values to the three people and that the logic correctly determines the one with the most apples.

  2. Design and Write the source code

    Moving across an X-Y grid

    Write a program that simulates moving across an X-Y grid. The program should start at the middle of the plot at position (0, 0) and keep track of the (X, Y) position. The tracker will move ten times, moving in one of four directions: up, down, left, or right. The program should display the new position after each move.

    Don't forget to add the required header information described in the previous assignments.

    Write a method called Move() and call it ten times. The method should take two parameters: the current X and Y position and return the new X and Y position as a tuple.

    You may use a loop to call the method if you prefer, or you can keep it very simple as I did in the example.

    The method should randomly select one of the four directions to move in, Up, Down, Left, or Right, and also randomly select the number of steps, between 1 and 10, to move in that direction. The method should then calculate and return the new X and Y position.

    Each time the method is called, display the direction and number of steps to move.

    Up, should increase the Y position by the number of steps, Down, should moved along the Y axis in the negative direction. Left and Right should move positive and negative along the X axis.

    Use the Pythagorean theorem to calculate the distance from the starting position. The formula is distance = sqrt((x2 - x1)^2 + (y2 - y1)^2).

    Write a method called letterGrade() that takes a single integer parameter and returns a string. The method should return the letter grade for the given number. The letter grades are as follows:
