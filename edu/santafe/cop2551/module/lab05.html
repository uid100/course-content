Lab 05: Exception Handling

Objective: Use try-catch-finally blocks to handle exceptions in C# code.

In this lab, you will practice using exception handling in C# code. You will learn how to use try-catch-finally blocks to handle exceptions and how to create custom exceptions.

As before, you will create a single solution with multiple projects. Each project will contain a separate exercise that demonstrates different aspects of exception handling.

Exercise #1: Safe Division with Exception Handling

  1. Create the project

    Lab05
    ├── Lab03.sln
    └── Division
        ├── Division.csproj
        └── Program.cs

    Objective

    Use a try-catch block to handle potential exceptions when dividing two numbers entered by the user, including handling division by zero and invalid input.

    Instructions

    1. Ask the user to enter two numbers: a numerator and a denominator.
    2. Convert the input to integers using int.Parse() (or Convert.ToInt32()).
    3. Perform division: numerator / denominator.
    4. Use try-catch to:
      1. Catch DivideByZeroException and display a message.
      2. Catch FormatException for invalid input and display a message.
      3. Catch any other exceptions and display a generic error message.
    5. Display a farewell message at the end using a finally block.

    Example Output

    Case 1: Valid Input 

            Enter numerator: 10
        Enter denominator: 2
        Result: 5
        Thank you for using the program!

    Case 2: Division by Zero 

            Enter numerator: 10
        Enter denominator: 0
        Error: Division by zero is not allowed.
        Thank you for using the program!

    Case 3: Invalid Input 

            Enter numerator: ten
        Enter denominator: 2
        Error: Invalid input. Please enter a valid number.
        Thank you for using the program!

    Don't forget to add the required header information.

  2. Explanation

    Understanding the code

    The code will use a try-catch block to handle exceptions that may occur during the division operation. The try block contains the code that may throw an exception, while the catch blocks handle specific exceptions that may occur.

    The first catch block should handle the DivideByZeroException, which occurs when the denominator is zero. The second catch block handles the FormatException, which occurs when the input cannot be converted to an integer. The third catch block handles any other exceptions that may occur. The order of these catch blocks is important, as the most specific exceptions should be caught first.

    The finally block is executed regardless of whether an exception occurs or not. It displays a farewell message to the user.

    The program should use int.Parse() to convert the input to an integer. If the input is not a valid integer, a FormatException will be thrown.

    The program also uses Console.ReadLine() to read input from the user. If the user enters a non-numeric value, a FormatException will be thrown.

  3. Run and test the program

    Run the program by clicking the green arrow (Start button) in the toolbar or by pressing Ctrl + F5. Or you can use either of the Start options in the Debug menu.

    Test the program with valid and invalid inputs.

Exercise #2: Overflowing integers

  1. Create the project

    From the solution folder (Lab05), create a new project and add it to the solution.

    • project name: Overflow
    • solution name: Lab05

    Objective

    Learn how an `OverflowException` can occur during arithmetic operations when a value exceeds the limits of a data type (e.g., `int.MaxValue`), and how to catch it.

    Instructions

    1. Use a `checked` block to enable overflow detection.
    2. Set an integer variable to `int.MaxValue`.
    3. Try adding 1 to the variable inside the `checked` block.
    4. Use a `try-catch` block to catch the `OverflowException` and display an error message.

    Example Output

            Attempting to add 1 to int.MaxValue...
        Overflow occurred! Value was too large for an int.

  2. Explanation

    Understanding the code

    The code will use a `checked` block to enable overflow detection. When an arithmetic operation exceeds the limits of the data type, an `OverflowException` will be thrown.

    The `try-catch` block will catch the `OverflowException` and display an error message. This allows the program to handle the exception gracefully instead of crashing.

  3. Run and test the program

    Test the program to ensure it correctly detects the overflow and displays the error message.

Exercise #3:

  1. Create the project

    From the solution folder (Lab05), create a new project and add it to the solution.

    • project name: MonthName
    • solution name: Lab05

    Objective

    Learn how to throw exceptions in C# and how to catch them in a try-catch block.

    Instructions

    1. Write a method monthName that:
      • Takes an integer month as a parameter.
      • Uses a switch expression to return the name of the month.
      • If the input is not between 1 and 12, throws an ArgumentOutOfRangeException.
    2. In the Main() method:
      • Ask the user to enter a month number (1–12).
      • Call the monthName() method with the user's input.
      • Catch and display any exceptions that occur.

    Example Output

    Case 1: Valid Input 

            Enter a month number (1-12): 5
        The month is: May

    Case 2: Invalid Input 

            Enter a month number (1-12): 13
        Error: Month must be between 1 and 12.

    Don't forget to add the required header information.

  2. Explanation

    Understanding the code

    The method monthName uses a switch expression to return the name of the month based on the input number. If the input is not between 1 and 12, it throws an ArgumentOutOfRangeException.

    The Main() method prompts the user for input, calls the monthName() method, and catches any exceptions that may occur.

  3. Run and test the program

    Run the program to ensure it correctly handles valid and invalid inputs.

    Test the program with various inputs, including valid month numbers (1–12) and invalid numbers (e.g., 0, 13, -1).
