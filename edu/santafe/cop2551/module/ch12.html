Introduction

Inheritance and polymorphism are two fundamental concepts in object-oriented programming (OOP) that promote code reusability and flexibility.

Class Inheritance (Is-a Relationship)

Inheritance allows one class (called a derived or child class) to inherit the properties and behaviors (fields and methods) of another class (called a base or parent class).

Imagine releasing a new product that is a variation of an existing company product. Instead of rewriting all the code, you can create a new class that inherits from the existing one, allowing you to reuse the code and add or override specific features. Yes, the new product (class) has new features, but it still retains the core functionality of the original product (class). In fact, many new products might evolve from existing products, sharing common -and already tested and proven- functionality. There is no need to start over from scratch.

The same is true for software development. Don't rewrite code that already works!

Key Points:

  • Use the : symbol to inherit from a base class.
  • Inherited members can be accessed as if they were part of the derived class.
  • Promotes code reuse and simplifies maintenance.

Example:

class Animal
        {
            public void Eat() => Console.WriteLine("Eating...");
        }

        class Dog : Animal
        {
            public void Bark() => Console.WriteLine("Barking...");
        }

Here, Dog inherits the Eat() method from Animal.

Polymorphism allows methods to behave differently based on the object that invokes them. It enables the same method name to perform different tasks.

Types:

  1. Compile-time polymorphism (method overloading)
  2. Run-time polymorphism (method overriding with virtual/override)

Example of Run-time Polymorphism:

class Animal
        {
            public virtual void Speak() => Console.WriteLine("Animal speaks");
        }

        class Dog : Animal
        {
            public override void Speak() => Console.WriteLine("Dog barks");
        }

        class Cat : Animal
        {
            public override void Speak() => Console.WriteLine("Cat meows");
        }

Animal myPet = new Dog();
        myPet.Speak();  // Output: Dog barks

Benefits:

  • Makes code more flexible and extensible
  • Enables dynamic behavior at runtime
Remember: Use virtual in the base class and override in the derived class to enable polymorphism.
