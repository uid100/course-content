Two-dimensional arrays (2D arrays) are a data structure that can be used to store data in a tabular format. Each element in a 2D array is associated with a row and a column number. This makes 2D arrays ideal for storing data that is naturally organized in rows and columns, such as spreadsheets, matrices, images, and game boards.

Working with 2D arrays in Java is relatively straightforward. The int[][] data type is used to declare a 2D array. The first dimension of the array represents the number of rows, and the second dimension represents the number of columns. Each element in the array can be accessed using its row and column numbers.

2D arrays are a powerful tool that can be used to solve a wide variety of problems in Java programming. They are especially useful for applications that involve data tables, image processing, and game development.

Here are some specific examples of how 2D arrays are used in Java programming:

  • Data tables: 2D arrays can be used to store data in a tabular format, such as a spreadsheet or a database table. This makes them ideal for applications that need to store and manipulate large amounts of data.

  • Image processing: 2D arrays can be used to store the pixels of an image. This allows Java programs to manipulate images by accessing and modifying individual pixels.

  • Game development: 2D arrays can be used to store the game board, the player's position, and the position of enemies and other objects in the game. This allows Java games to be rendered and updated efficiently.

As mentioned, 2D arrays are a data structure that can be used to store data in a tabular format. This makes them ideal for storing images, which are naturally organized in rows and columns of pixels. A pixel is the smallest unit of an image. It is a tiny dot that is assigned a color value. The color value of a pixel is typically represented by three numbers, one for each of the primary colors: red, green, and blue (RGB).

 

In a digital image, the pixels are arranged in a 2D grid. The number of rows and columns in the grid determines the resolution of the image. The higher the resolution, the more pixels there are in the image, and the more detail the image can show. The pixels in an image can be stored in a 2D array. The first dimension of the array represents the rows of pixels, and the second dimension represents the columns of pixels. Each element in the array stores the color value of a single pixel.

For example, a 100x100 image with 24-bit color depth (which means each pixel can have 256 possible colors) would be stored in a 2D array of size 100x100x24. The first dimension of the array would represent the 100 rows of pixels, the second dimension would represent the 100 columns of pixels, and the third dimension would represent the 24 bits of color information for each pixel.

By storing images in 2D arrays, Java programs can access and manipulate individual pixels quickly and efficiently. This makes it possible to perform a variety of image processing tasks, such as resizing, cropping, rotating, and filtering images. Pixel.java

A pixel is the smallest unit of a display, in this case, of a picture. A picture is made up of hundreds, thousands, or millions of pixels to make up the entire image. For example, a 4K image with a dimension of 3840 x 2160 has 8,294,400 pixels.

  • A pixel has a color made up of the 3 colors red, green, and blue

  • These colors can be retrieved using their corresponding getters: getRed, getGreen, getBlue. An example is shown.

    int greenValue = p.getGreen(); //assuming p is a Pixel object

  • These color values can also be set using their corresponding setters: setRed, setGreen, setBlue. For example:

    p.setGreen(40);

  • The range of values for each color channel, red, green, and blue, is between 0 and 255 inclusive.

    0 means no amount of color, and 255 is the maximum amount.

makeSunset

This filter can enhance the red levels of a picture while decreasing the green and blue values creating a sunset filter effect.

  1. Get the RGB value of the pixel.
  2. Calculate new values for red by multiplying the red value of all the pixels by the multiplier redIntensity to "enhance" the red (since the multiplier is > 1.0 ).
  3. Calculate new values for green, and blue by multiplying the the current values by the multiplier, greenBlueReduction, to decrease their value (since the multiplier is > 1.0 ).
  4. Update the pixel color values with the calculated values from steps 2 & 3.

blur

This is a more challenging filter as you must consider a rectangular window that will move throughout the image.

  1. Take a 2x2 window (starting from the upper-left corner of the image) and sum all the red, green, and blue values separately for all the pixels in that window. In a 2-by-2 window, there will be a total of 4 pixels for which to sum the three separate color channels.
  2. Take the sum found for each channel of all the pixels in the window and divide them by 4. (4 pixels in the combined, blurred area.)
  3. Set the red, green, and blue values of the top-left-most pixel of the 2-by-2 window to be the value calculated from #2.
  4. Perform this on the entire image moving the 2x2 window by one pixel each time from left-to-right and top-to-bottom where top-most will be row 0 and left-most is column 0.
