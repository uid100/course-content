Create Lab Folders

The first part of the lab is to set up your folder structure. Take a few minutes to create a folder structure for your labs like the one shown. Being organized is a critical survival skill in software development.

If you are using the classroom lab computers, be sure to create this on your USB drive and be sure to save to it! Saving on the C: drive of the lab computers is a disaster waiting to happen!

Many systems traditionally don't allow using spaces in file (and folder) names. This is a good practice for all of your files, but especially when writing code. Lab01 is not the same as Lab 01 or Lab assignment 1. When you start processing multiple data files, using spaces in file names will cause unexpected problems. Learn to avoid that now.

There are 16 weeks and 12 Labs in this class. If you name your folders: Lab1, Lab2, Lab3, ... then your files will sort themselves as Lab1, Lab10, Lab11, Lab12, Lab2, Lab3... I suppose this is ok, but it might be frustrating when you're looking for a file, so since there are more than 9 and less than 100 labs, using two digits for the folder name makes sense.

When you submit your file (and it is called Lab01.zip) it unzips to the same folder structure that I'm expecting, and it's easy for me to find, and shows that you read and followed the instructions.

Windows or Mac?

This is a question that I get asked frequently. Python is supported across several platforms. I program on different platforms and I regularly test solutions on a Windows PC, a Macbook, and a Linux computer. Many of the examples have even been tested on my iPhone!

For this class, either Windows or MacOS will work just fine, but you should choose the one that you are most comfortable with. To be successful, programming experience is not required, but knowledge and comfort with your computer is.

Filename Extensions

Please don't skip this step!

Before we go any further, make sure that the setting to view filename extensions is enabled. The exact location or method to check this will vary between systems and even from release to release, but in general, this should work.

For Windows, open the File Explorer and find the View menu. Set the option to Show > File Extensions. The exact location of this setting has changed between the last few versions of Windows, but this should be enough to find it.

Mac users, open the Finder and select Preferences and check Show all filename extensions

Exercise 1 - Multiply 2 Numbers
Guided Python Application

For most of the labs in this class, the first exercise will be a guided walk-through where we will apply the chapter concepts to develop a working Python application.

In this exercise, you will be guided through the steps of analyzing a problem statement, writing pseudocode, and creating a working Python application using Python IDLE (Integrated Development and Learning Environment). Be sure to complete this exercise as you will be repeating these steps and writing the same (or very similar) code for the other exercises.

Be aware, practice is the key here. The more you do, the easier it gets!

This course is about program logic and design. Specifically, that means the approach to the problem is as important as the final solution! To say that another way, the program design is as important as the program.

Step one is always to analyze the problem and figure out a plan of attack. To save time, we'll continue the problem we l ooked at in the chapter one lecture notes and the final problem statement is pasted below for reference. Please review the chapter one notes if you need to review the more detailed analysis and pseudocode.

Problem statement from chapter one notes: It's a bright, sunny day and Cuyamaca Learning Center has hired you as a consultant to develop an application that will accept two input numbers from the user and display the product> of the two values (numbers.) Notice that the first part of that statement isn't relevant to the solution, so we can ignore it.

Break it down into

  1. input,
  2. process, and
  3. output
This is a simple problem and we can easily break it down into “workable pieces” using this approach.
  • What input do we need? We need 2 numbers
  • What is our process? We need to multiply the 2 numbers.
    Our formula is: product = number1 * number2
  • What is our output: We need to display the product
